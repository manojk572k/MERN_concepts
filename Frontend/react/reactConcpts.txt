🚀 When does render happen?
1. Initial load
2. State change (setText)
3. Props change
4. Parent re-render
“Render means executing the component function to compute JSX. It doesn’t always mean updating the DOM — React compares and updates only what changed.”

<Suspense> lets you display a fallback until its children have finished loading.
<Suspense fallback={<Loading />}>
  <SomeComponent />
</Suspense>
“Suspense allows React to delay rendering of a component and show a fallback UI until the component or resource is ready.”

Activity> lets you hide and restore the UI and internal state of its children.
<Activity mode={visibility}>
  <Sidebar />
</Activity>
It is useful when I want to hide something that is likely to come back soon, but I do not want to lose local UI state, such as:
tab panels
sidebars
drafts in forms
expandable panels
pre-rendered content likely to become visible soon.
Important caveat
If the hidden content has useEffect subscriptions, timers, or listeners, React will clean those up while hidden and recreate them when shown again. So preserved state/DOM does not mean effects keep running in the background.
Simple mental model
{show && <Comp />} → hide by unmounting → state lost
<Activity mode="hidden"><Comp /></Activity> → hide by preserving → state kept, effects cleaned up.

🔹 Controlled vs Uncontrolled (Short)
✅ Controlled
Value stored in React state
Uses: value + onChange
React controls everything
👉 Best for forms, validation
✅ Uncontrolled
Value stored in DOM
Uses: ref
React reads value only when needed
👉 Best for simple / fast inputs
🔹 One-line memory

Controlled = state
Uncontrolled = ref

🔹 Interview line

“Controlled components use React state as the source of truth, while uncontrolled components rely on the DOM.”


🔹 React Rendering (Short)
Rendering = state/props → UI
React re-renders when:
State changes
Props change
Parent re-renders
🔹 How it works

State change → New Virtual DOM → Compare → Update only changed parts

🔹 Key concept

👉 Virtual DOM = copy of real DOM for efficient updates

🔹 Important
Re-render ≠ DOM update
React updates only what changed
🔹 One-line memory

“State changes → React recalculates UI → updates minimal DOM”


🧠 What is “Lifting State Up”?
👉 Move state to the closest common parent
so multiple components can use it.

Move state to the closest common parent
Used when multiple components need the same data

Children communicate changes using function props
